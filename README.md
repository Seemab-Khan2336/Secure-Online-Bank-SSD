# Online Secure Bank: Secure Software Development 🔐

The **Online Secure Bank** project is a security-first banking application developed to address critical vulnerabilities in traditional financial systems. Built using a **Secure SDLC** approach, the system integrates defensive coding and systematic threat modeling from the design phase.

## 🛡️ Key Security Features
* **Robust Authentication:** Implementation of secure login mechanisms to prevent unauthorized access.
* **Advanced Data Protection:** Utilizes **SHA-256 hashing** to safeguard sensitive user credentials and transaction data.
* **STRIDE Threat Modeling:** Systematically identifies and mitigates threats including Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, and Elevation of Privilege.
* **Integrity Validation:** Comprehensive unit testing framework to ensure system reliability and security logic.

## 🛠️ Tech Stack & Tools
* **Programming:** Python / Java (Logic Implementation)
* **Hashing:** SHA-256
* **Threat Modeling:** Microsoft STRIDE Framework
* **Testing:** Alloy / Unit Testing Frameworks
* **Design:** Draw.io (DFDs and System Architecture)

## 📂 Project Structure
* `Report__Project__SSD.docx`: Detailed technical report covering the security architecture and STRIDE mapping.
* `PROJECT PROPOSAL_SSD.docx`: Initial scope, security requirements, and project objectives.
* `Online-Secure-Bank.pptx`: Presentation slides highlighting critical security gaps and implementation results.
